All the following code is to be run in a canvas tag. As we inspect the code we will see how we are targetting the Canvas tag element using the DOM in order to manipulate it and inject the chart from our CDN. 


const ctx0 = document.getElementById("introChart"); //use DOM to target the HTML element canvas
ctx0.getContext("2d");

/*In this case I added a body of options that is nothing else but an object.
NOTE: chart.js has a specific order for the properties of the object to be.*/

const myOptions = { 
    responsive: true,
    maintainAspectRatio: false,
    scale: {
    angleLines: {
        display: true,
    },
    },
    ticks: {
    //   min: 0,
    max: 20,
    beginAtZero: true,
    stepSize: 5,
    },
};

I start instantiating my variables as a good practice. We will be making use of that object as a whole value that we will be able to tweak to suit our needs. 


/*IMPORTANT: Naming conventions are of great importance in Chart.js. 
the constant name of the chart has to be the same of the canvas id */

const introChart = new Chart(ctx0, { 
type: "radar", //type is the value where we define the type of chart.

/*data takes values such as labels and datasets 
    (this allows us to add more datasets dynamically) isnt that exiting? */

data: { 
labels: [
    "Kung-Fu",
    "Dog-poop scooping",
    "Skateboarding",
    "Motorcycle riding",
    "Programming",
    "Languages",
],
datasets: [
    {
    label: "Skills Matrix", //label name of my chart
/*'data' takes an array with values & 
    the library takes care of pair those values to the labels! */
        data: [16, 19, 12, 16, 18, 13],
        backgroundColor: "rgba(54, 162, 235, 0.2)", //colors to be added use RGB values
        borderColor: "blue",  //but it also takes strings for colors
        borderWidth: 2,
      },
    ],
  },
options: myOptions, //I made use of the object constant previously instanciated.
});
